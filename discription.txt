---

## 🎯 VUE D'ENSEMBLE DU PROJET

### Nom du Projet
**P2M-FTTH Infrastructure Inventory & Monitoring Dashboard**

### Description
Plateforme web de supervision en temps réel des réseaux fibres optiques FTTH (Fiber To The Home) avec intelligence artificielle pour la prédiction de pannes. Le système permet de monitorer l'état des RTU (Remote Terminal Units), détecter automatiquement les anomalies, et prédire les pannes potentielles 24-48h à l'avance.

### Objectifs Principaux
1. **Supervision temps réel** : Monitoring continu de 50-100 RTU
2. **Détection automatique** : Alarmes basées sur seuils ITU-T
3. **Prédiction IA** : Anticipation des pannes avec Random Forest (accuracy >75%)
4. **Visualisation** : Dashboard intuitif avec graphiques temps réel
5. **Maintenance prédictive** : Réduction MTTR et optimisation interventions

## 📊 KPIs (INDICATEURS DE PERFORMANCE)

### 1. Disponibilité Réseau

**Formule :**
```
Disponibilité (%) = (Uptime / Total Time) × 100
```

**Calcul :**
```sql
SELECT 
  COUNT(CASE WHEN status = 'online' THEN 1 END)::DECIMAL / COUNT(*) * 100 
  AS availability_percent
FROM rtu;
```

**Target :** 99.9% (norme ARCEP)
**Signification :** 
- 99.9% = 8.76h downtime/an acceptable
- <99% = Problème grave réseau

**Affichage dashboard :** Widget avec jauge circulaire

---

### 2. MTTR (Mean Time To Repair)

**Formule :**
```
MTTR = Σ(temps résolution) / nombre alarmes résolues
```

**Calcul :**
```sql
SELECT 
  AVG(EXTRACT(EPOCH FROM (resolved_at - timestamp))/3600) AS mttr_hours
FROM alarms
WHERE resolved = TRUE
  AND resolved_at > NOW() - INTERVAL '30 days';
```

**Target :** <4 heures
**Signification :**
- <2h = Excellent
- 2-4h = Bon
- >4h = À améliorer

**Affichage dashboard :** Widget avec tendance

---

### 3. Alarmes par Sévérité

**Requête :**
```sql
SELECT 
  severity,
  COUNT(*) as count
FROM alarms
WHERE resolved = FALSE
GROUP BY severity;
```

**Classification :**
- **CRITICAL** : Panne totale, coupure fibre (>18 dB, RTU offline)
- **MAJOR** : Dégradation sévère (13-18 dB, température >40°C)
- **MINOR** : Dégradation légère (10-13 dB, température >30°C)
- **INFO** : Informationnel (tests, changements config)

**Affichage dashboard :** Widget avec badges colorés

---

### 4. Taux de Perte Optique

**Formule :**
```
Loss Rate (%) = (Atténuation mesurée - Atténuation référence) / Atténuation référence × 100
```

**Calcul :**
```sql
SELECT 
  AVG(attenuation_db) as avg_attenuation,
  MAX(attenuation_db) as max_attenuation
FROM test_results
WHERE timestamp > NOW() - INTERVAL '24 hours';
```

**Seuils (norme ITU-T G.652) :**
- 10-12 dB : Normal
- 13-17 dB : Dégradé
- >18 dB : Critique

---

### 5. Prédictions IA - Accuracy

**Formule :**
```
Accuracy = (Vrais Positifs + Vrais Négatifs) / Total Prédictions
```

**Calcul :**
```sql
SELECT 
  COUNT(CASE WHEN was_correct = TRUE THEN 1 END)::DECIMAL / COUNT(*) * 100
  AS accuracy_percent
FROM predictions
WHERE was_correct IS NOT NULL;
```

**Target :** >75% (minimum projet)
**Actuel (v2.0) :** 85.23%

---

### 6. RTU par Statut

**Requête :**
```sql
SELECT 
  status,
  COUNT(*) as count,
  (COUNT(*)::DECIMAL / (SELECT COUNT(*) FROM rtu) * 100) as percentage
FROM rtu
GROUP BY status;
```

**Répartition cible :**
- Online : >85%
- Warning : <10%
- Offline : <5%

## 🤖 INTELLIGENCE ARTIFICIELLE

### Algorithme : Random Forest Classifier

**Librairie :** scikit-learn (Python)

**Architecture :**
```
Forêt aléatoire de 200 arbres de décision
├── Arbre 1 (max_depth=15)

       ↓
Vote majoritaire → Prédiction finale
```

### Features (8 Variables d'Entrée)

**1. attenuation_mean_7d** (float)
- Description : Atténuation moyenne des 7 derniers jours
- Unité : dB (décibels)
- Plage : 10.0 - 25.0
- Source : Moyenne de `test_results.attenuation_db`

**2. nb_alarms_24h** (int)
- Description : Nombre d'alarmes dans les dernières 24h
- Plage : 0 - 20
- Source : COUNT sur `alarms` WHERE timestamp > NOW() - 24h

**3. nb_alarms_7d** (int)
- Description : Nombre d'alarmes dans les 7 derniers jours
- Plage : 0 - 50
- Source : COUNT sur `alarms` WHERE timestamp > NOW() - 7d

**4. uptime_percent** (float)
- Description : Pourcentage de disponibilité
- Plage : 85.0 - 100.0
- Formule : (temps online / temps total) × 100

**5. temperature** (float)
- Description : Température actuelle du RTU
- Unité : °C
- Plage : 15.0 - 55.0
- Source : Dernière valeur `test_results.temperature`

**6. attenuation_variation** (float)
- Description : Écart-type atténuation sur 7 jours
- Unité : dB
- Plage : 0.0 - 5.0
- Formule : STDDEV(`attenuation_db`)

**7. rtu_age_days** (int)
- Description : Âge du RTU en jours depuis installation
- Plage : 0 - 3650 (10 ans max)
- Formule : (NOW() - installation_date) en jours

**8. fiber_type** (int)
- Description : Type de fibre
- Valeurs : 
  - 0 = Multi-mode
  - 1 = Single-mode (majoritaire FTTH)

---

### Processus de Prédiction

**1. Collecte Features (Toutes les 6h via Cron)**

```python
def collect_features(rtu_id):
    # Récupérer historique RTU
    tests = db.query("SELECT * FROM test_results WHERE rtu_id = ? ORDER BY timestamp DESC LIMIT 100", rtu_id)
    alarms = db.query("SELECT * FROM alarms WHERE rtu_id = ? AND timestamp > NOW() - INTERVAL '7 days'", rtu_id)
    
    features = {
        'attenuation_mean_7d': np.mean([t['attenuation_db'] for t in tests[:14]]),
        'nb_alarms_24h': len([a for a in alarms if a['timestamp'] > NOW() - timedelta(hours=24)]),
        'nb_alarms_7d': len(alarms),
        'uptime_percent': calculate_uptime(rtu_id),
        'temperature': tests[0]['temperature'],
        'attenuation_variation': np.std([t['attenuation_db'] for t in tests[:14]]),
        'rtu_age_days': (NOW() - rtu['installation_date']).days,
        'fiber_type': 1  # Single-mode
    }
    
    return features
```

**2. Normalisation**

```python
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
features_normalized = scaler.fit_transform([features.values()])
```

**3. Prédiction**

```python
# Charger modèle v2.0
model = joblib.load('/models/model_v2.pkl')

# Prédire probabilité
probability = model.predict_proba(features_normalized)[0][1]  # Classe 1 = panne

# Déterminer risk_level
if probability > 0.85:
    risk_level = 'critical'
elif probability > 0.70:
    risk_level = 'high'
elif probability > 0.50:
    risk_level = 'medium'
else:
    risk_level = 'low'
```

**4. Stockage BDD**

```python
db.insert("predictions", {
    'rtu_id': rtu_id,
    'probability': round(probability, 4),
    'risk_level': risk_level,
    'prediction_horizon_hours': 48,
    'model_id': 3,  # v2.0

    'valid_until': NOW() + timedelta(hours=48),

```

if risk_level in ['high', 'critical']:

## 🔔 SYSTÈME D'ALARMES

### Détection Automatique

**Algorithme de détection :**

```python
def detect_alarm(reference_trace, current_trace):
    """
    Compare trace référence vs mesure actuelle
    Retourne alarme si seuils dépassés
    """
    
    # 1. Calculer différence atténuation
    delta_db = current_trace.attenuation - reference_trace.attenuation
    
    # 2. Classification par seuils (norme ITU-T G.652)
    if delta_db > 6.0:  # >6 dB de différence
        severity = "CRITICAL"
        alarm_type = "fiber_cut"  # Probable coupure fibre
        message = f"Coupure fibre probable : delta +{delta_db:.1f} dB"
        
    elif delta_db > 3.0:  # 3-6 dB
        severity = "MAJOR"
        alarm_type = "high_loss"
        message = f"Perte optique élevée : delta +{delta_db:.1f} dB"
        
    elif delta_db > 1.5:  # 1.5-3 dB
        severity = "MINOR"
        alarm_type = "degradation"
        message = f"Dégradation fibre : delta +{delta_db:.1f} dB"
        
    else:
        return None  # Pas d'alarme
    
    # 3. Vérifier température RTU
    if current_trace.temperature > 50:
        severity = "CRITICAL"
        alarm_type = "high_temperature"
        message = f"Température critique : {current_trace.temperature}°C"
        
    elif current_trace.temperature > 40:
        severity = "MAJOR"
        alarm_type = "temperature_warning"
        message = f"Température élevée : {current_trace.temperature}°C"
    
    # 4. Créer alarme
    return Alarm(
        severity=severity,
        alarm_type=alarm_type,
        message=message,
        delta_db=delta_db
    )
```

---

### Seuils de Détection (Normes Industrielles)

**Atténuation (ITU-T G.652) :**
```
Normal :     10-12 dB  (fibre saine)
Dégradé :    13-17 dB  (surveillance accrue)
Critique :   >18 dB    (intervention requise)
```

**Température RTU (TIA-568) :**
```
Normal :     -5 à 40°C
Warning :    40 à 50°C
Critical :   >50°C
Shutdown :   >60°C (arrêt automatique)
```

**Power Optique (IEEE 802.3) :**
```
Normal :     -5 à -12 dBm
Low :        -13 à -20 dBm
Critical :   <-20 dBm
```

**Reflectance (ITU-T G.650) :**
```
Excellente : <-55 dB
Bonne :      -50 à -55 dB
Acceptable : -45 à -50 dB
Mauvaise :   >-45 dB (mauvaise soudure/connecteur)
```

## 🔒 SÉCURITÉ

### Authentification JWT

```javascript
// Génération token
const token = jwt.sign(
  { id: user.id, username: user.username, role: user.role },
  process.env.JWT_SECRET,
  { expiresIn: '24h' }
);

// Middleware validation
function authMiddleware(req, res, next) {
  const token = req.headers.authorization?.split(' ')[1];
  
  if (!token) {
    return res.status(401).json({ error: 'No token provided' });
  }
  
  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET);
    req.user = decoded;
    next();
  } catch (error) {
    return res.status(401).json({ error: 'Invalid token' });
  }
}
```
